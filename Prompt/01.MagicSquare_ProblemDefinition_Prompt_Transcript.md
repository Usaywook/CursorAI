## STEP 1 — Observation (관찰)

### User Prompt

```
--------------------------------------------
STEP 1 — Observation (관찰)
--------------------------------------------
다음 질문에 답하십시오:
1. 현재 우리가 해결하려는 상황은 무엇인가?
2. 왜 4x4 마방진 문제를 다루는가?
3. 이 문제는 어떤 학습 또는 시스템 설계 맥락에서 등장하는가?
"마방진을 만든다"는 표현 대신, 상황을 관찰 관점으로 서술하십시오.
```

### AI 응답 핵심 요약

**Q1 — 상황:**
부분적으로 채워진 4×4 격자가 주어졌을 때, 비어 있는 두 칸에 어떤 숫자를 넣어야 모든 행·열·대각선의 합이 동일해지는가? — 탐색 + 검증의 복합 상황.

**Q2 — 왜 마방진인가:**

| 관점 | 내용 |
|---|---|
| 규칙의 명확성 | 합=34 불변식이 수학적으로 정해져 있어 테스트 기준 명확 |
| 입출력 계약 단순함 | 4×4 격자 → 6개 정수 경계 분명 |
| 복잡성 적절함 | 설계 연습에 적합한 난이도 |
| 실패 시나리오 다양성 | 해 없음·입력 오류·중복 등 예외 풍부 |

**Q3 — 학습 맥락:** Cursor AI 보조 개발 학습 / Dual-Track TDD 실습 / ECB 아키텍처 설계 연습

### 완료 조건

- [x] "채우기"가 아닌 "탐색+검증의 복합 상황"으로 서술됨
- [x] 4×4 규모 선택 이유 명시됨
- [x] 학습·설계 맥락이 3가지로 분리되어 서술됨

## STEP 2 — Why #1

### User Prompt

```
--------------------------------------------
STEP 2 — Why #1
--------------------------------------------
Q: 왜 마방진을 완성해야 하는가?
A를 가정하여 답하고, 그 답에서 드러나는 불편함 또는 구조적 문제를 분석하십시오.
```

### AI 응답 핵심 요약

**가정 답변 A:**
"사용자가 입력한 부분 격자가 올바른 마방진이 되는지 확인하고, 빈 칸에 들어갈 정답을 알려줘야 하기 때문이다."

**드러나는 구조적 문제 — 세 가지 행위 혼재:**

| 행위 | 숨겨진 문제 |
|---|---|
| ① 검증 — "올바른지 확인한다" | 기준이 암묵적 (행만? 열과 대각선도?) |
| ② 탐색 — "빈 칸 위치를 찾는다" | 빈 칸이 0·1·3개 이상이면 처리 미정 |
| ③ 계산 — "정답을 알려준다" | 해가 유일한가? 없으면? 형식은? |

**핵심 발견:**
"완성한다"는 단어 하나에 검증·탐색·계산이 압축되어 있어, 각 책임의 경계가 명확하지 않다.

### 완료 조건

- [x] 가정 답변 A 제시됨
- [x] 3가지 행위 혼재 발견됨
- [x] 각 행위별 숨겨진 문제 분석됨

## STEP 3 — Why #2

### User Prompt

```
--------------------------------------------
STEP 3 — Why #2
--------------------------------------------
Q: 왜 단순 계산이 아니라 프로그램으로 구현하는가?
다음 관점에서 답하십시오:
- 반복 가능성
- 검증 자동화
- 오류 방지
- 규칙 기반 사고 훈련
```

### AI 응답 핵심 요약

**① 반복 가능성:**
격자가 바뀌어도 동일한 절차 재호출 → 손 계산은 매번 처음부터.

**② 검증 자동화:**
4행 + 4열 + 2대각선 = 총 10번의 덧셈을 한 번의 함수 호출로 처리. 검증 로직 자체도 테스트로 검증 가능.

**③ 오류 방지:**

| 오류 유형 | 프로그램 처리 |
|---|---|
| 입력 오류 (17, 중복, 빈칸 3개) | 입력 검증 단계에서 즉시 거부 |
| 계산 오류 (조합 누락) | Step A + Step B 모두 시도 |

**④ 규칙 기반 사고 훈련:**

| 모호한 직관 | → | 명확한 규칙 |
|---|---|---|
| "대충 맞는 것 같다" | | "모든 합 = 34, 1~16 중복 없음" |
| "당연히 빈 칸은 2개" | | I1: 빈 칸은 정확히 2개 |
| "답이 8과 1이다" | | "Step A인가 Step B인가?" |

**핵심:** 프로그램 구현은 사고의 허점을 코드가 드러내게 만드는 훈련이다.

### 완료 조건

- [x] 4가지 관점 모두 서술됨
- [x] "규칙 강제 번역"이 핵심으로 도출됨

## STEP 4 — Why #3 (핵심 발견)

### User Prompt

```
--------------------------------------------
STEP 4 — Why #3 (핵심 발견)
--------------------------------------------
Q: 왜 이 문제를 TDD 방식으로 설계하려 하는가?
- 무엇이 통제되어야 하는가?
- 어떤 불변 조건이 존재하는가?
- 왜 입력/출력이 명확해야 하는가?
```

### AI 응답 핵심 요약

**무엇이 통제되어야 하는가 — 4곳의 오류 지점:**

```
[입력] → [검증] → [탐색] → [계산] → [출력]
          ↑          ↑         ↑         ↑
        오류①      오류②     오류③     오류④
```

| 오류 지점 | 실패 예시 | 통제 방법 |
|---|---|---|
| ① 검증 | 빈 칸 3개 통과 | 입력 검증 테스트 |
| ② 탐색 | 0-index vs 1-index 혼용 | 탐색 함수 단위 테스트 |
| ③ 계산 | Step A만 시도 | 두 조합 검증 테스트 |
| ④ 출력 | 좌표와 값 순서 뒤바뀜 | 출력 계약 테스트 |

**불변 조건:**
"빈 칸이 정확히 2개여야 한다"는 수학이 아니라 **설계 결정**이다.
TDD는 이 결정을 테스트 코드에 박제한다.

**왜 입출력이 명확해야 하는가:**
```
입력 명확 → Boundary(입력 검증)가 무엇을 막아야 할지 안다
출력 명확 → Control(계산)이 무엇을 반환해야 할지 안다
둘이 명확 → Entity(도메인 규칙)가 어디까지 책임지는지 안다
```
입출력이 모호하면 테스트를 쓸 수 없다 → 구현이 맞는지 알 수 없다.

**핵심 발견:**
입출력 계약과 불변식이 먼저 정의되어야 구현이 가능하다.

### 완료 조건

- [x] 4개 오류 지점 도출됨
- [x] 불변식이 "설계 결정"임을 명시함
- [x] 입출력 계약과 레이어 분리의 연결 관계 서술됨

## STEP 5 — 진짜 문제 정의

### User Prompt

```
--------------------------------------------
STEP 5 — 진짜 문제 정의
--------------------------------------------
다음을 작성하십시오:
1. 표면 문제 정의 (잘못된 정의)
2. 개선된 문제 정의 (정확한 정의)
3. 이 문제의 핵심 Invariant
4. 우리가 실제로 훈련하려는 사고 능력
- 구현 설계는 하지 마십시오.
- 코드 언급 금지.
- 알고리즘 설명 금지.
- 구조화된 Markdown으로 작성하십시오.
```

### AI 응답 핵심 요약

**표면 문제 정의 (잘못된):**
"4×4 마방진에서 비어 있는 두 칸을 찾아서 채워 넣는 프로그램을 만든다."
→ 결과 중심 / 전제 숨김 / 경계 없음 / 책임 뭉침

**개선된 문제 정의:**
"부분적으로 채워진 4×4 격자를 입력받아, 정해진 조건을 만족하는지 검증한 뒤,
두 개의 빈 칸에 들어갈 값을 결정하여, 그 위치와 값을 정해진 형식으로 응답하는 시스템."

**핵심 Invariant:**

| 분류 | ID | 불변식 |
|---|---|---|
| 입력 | I1 | 격자는 반드시 4행 4열 |
| 입력 | I2 | 빈 칸(0)은 정확히 2개 |
| 입력 | I3 | 빈 칸 외 값은 1~16 |
| 입력 | I4 | 빈 칸 외 값은 중복 없음 |
| 출력 | I5 | 응답은 [r1,c1,v1,r2,c2,v2] 6개 정수 |
| 출력 | I6 | 위치는 1-index |
| 출력 | I7 | 완성 격자의 모든 합 = 34 |
| 해 결정 | I8 | 값은 격자에 없는 숫자에서만 선택 |
| 해 결정 | I9 | 두 배치 순서 모두 시도 후 성립하는 것 선택 |
| 해 결정 | I10 | 해가 없으면 명시적으로 알림 |

**훈련하려는 사고 능력:**

| 능력 | 내용 |
|---|---|
| ① 전제를 발견하는 능력 | "항상 성립한다고 가정한 것"을 먼저 묻는 능력 |
| ② 책임을 분리하는 능력 | 하나의 행위를 검증/탐색/결정/응답으로 쪼개는 능력 |
| ③ 실패를 먼저 상상하는 능력 | 비정상 경로를 정상 경로보다 먼저 정의하는 능력 |
| ④ 계약을 명문화하는 능력 | "그냥 안다"가 아닌 검증 가능한 명제로 남기는 능력 |
| ⑤ 관찰에서 정의로 이행하는 능력 | "무엇인지 안다"에서 "무엇을 보장해야 하는가"로 스스로 이행 |

**최종 문제 정의문:**
> 이 시스템은 부분적으로 채워진 4×4 격자를 입력받아,
> I1~I4의 입력 불변식을 검증하고,
> I8~I9의 해 결정 원칙에 따라 두 빈 칸의 값을 결정하며,
> I5~I7의 출력 불변식을 만족하는 형식으로 응답한다.
> 어떤 불변식도 위반될 경우, 시스템은 그 이유를 명시적으로 알린다.

### 완료 조건

- [x] 표면 정의와 개선 정의의 차이 명시됨
- [x] 불변식 I1~I10 도출됨
- [x] 5가지 훈련 사고 능력 도출됨
- [x] 코드·알고리즘 없이 서술됨
